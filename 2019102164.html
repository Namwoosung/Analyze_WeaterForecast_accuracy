Termproject¶

주제 선정 이유¶

우리의 생활과 밀접한 관련을 가지고 있는 것중 하나가 날씨이다. 이러한 날씨를 분석하고 예측해 우리에게 예보주는 것이 기상청의 역할이다.
그러나 예전부터 최근까지도 이런한 기상청에 대한 비판은 끊이지 않는다. 실제로 일상생활을 하다보면 맑음이라고 예보된 날 비가 와서 급히 우산을 사거나, 비가 온다고 해서 우산을 챙겼지만 비가오지않아 짐이 늘어난 경험은 다들 한번쯤 해봤을 것이다.
그렇기에 나는 이번 프로젝트에서 이렇게 일상생활과 밀접한 관계를 가지고 있는 날씨를 예보하는 기상청의 정확도를 측정해 보기위해 과거 기상청의 일기예보에서 예측한 강수확여부과 실제 강수횟수에 대한 데이터를 수집하고 비교해 기상청의 일기예보의 정확성을 판단해보려한다.

가설 정의¶

이번 프로젝트에서는 지역을 영통구로 한정하고 기간은 2016년~2018년 3년을 설정하여 3년동안의 강수예측과 강수여부에 대한 데이터를 분석하여 정확성을 판단해보려한다. 여기서 비, 눈은 구분하지않고 강수의 여부만 판단하겠다.
그동안 기상청의 예보실패로 인한 비판을 들은경험과 나의 실제 잘못된 기상예보를 접했던 경험을 토대로 하였을때 기상청의 강수여부 예보의 정확성은 70%이하일 것으로 추정된다.
가설: 최근 5년간 영통지역에서 기상청의 강수여부 예보의 정확성은 70%이하일 것이다.

#실제 일별 강수형태관측 결과를 얻기위해 앞서 획득한 csv파일을 가공하여 정리

import csv
#csv파일을 불러오기위해 csv모듈을 import

location = ['maetan1','maetan2','maetan3','maetan4','woncheon','yeongtong1','yeongtong2','taejang','gwanggyo1','gwanggyo2']
year = ['2016','2017','2018']
rain = {}
#수집해야하는 데이터가 영통구의 각 동10개와 2016년부터 2018년 까지의 데이터이므로 이를 불러들이기 위해 location리스트와 year리스트를 만듦
#실제 강수형태 관측데이터를 저장할 수 있도록 rain이라는 빈 딕셔너리를 만듦
for i in location:
    for j in year:
        data_lst=[]
        information = open('{0}_{1}.csv'.format(i,j),'r')
        lines = csv.reader(information)
        #반복문을통해 실제 관측데이터 csv파일을 차례로 획득
        #csv파일에서 획득할 데이터를 지역과 년도에 나누어 임시로 저장할 공간이 필요하므로 data_lst라는 빈 리스트를 만듦
        k = 0
        #변수 k는 이후에 데이터 정보를 보기용이하게 만들 때 사용
        for line in lines:
        #csv파일 각 행을 하나씩 입력
            del line[1]
            #각 행의 두번째 열 정보인 시간정보는 이번 프로젝트에서 필요하지 않으므로 삭제
            if line[1] == '0' or line[0] == ' ':
                continue
            #데이터의 양이 너무 많으므로 맑음(0)으로 관측된 날의 날씨정보를 추출하지않음, 빈 데이터또한 추출하지않음
            else :
                data_lst.append(line)
            #관측결과 강수가 있었던 날의 정보를 data_lst안에 입력
            if 'Start' in line[1] or 'Start' in line[0]:
                k = k + 1
                data_lst.remove(line)
            else:
                line[0] = str(int(('{0}'.format(j))+str(k).zfill(2)+'01')+int(line[0])-1)
            #csv파일안 날짜의 일만 알려주던 첫번째 정보를 년.월.일모두 알려주도록 편집
            for test in data_lst:
                num_test = data_lst.count(test)
                if num_test>1:
                    for a in range(num_test-1):
                        data_lst.remove(test)
            data_lst.sort()
            #편집을 통해 데이터를 간략화,유연화하는 과정에서 발생한 동일한 데이터들을 제거, 남은 데이터를 정렬
            rain[i+'_'+j]=data_lst
            #처리된 데이터를 rain이라는 딕셔너리에 입력
        information.close()
        #csv파일을 닫음

incorrect = 0
full_date = 365 * 3 * 10
#전체 날짜수 정보를 넣기 위해 full_date변수를 생성(여기서 총 3년간 10개지역의 정보를 추출하였으므로 365*3*10의 값을 대입함)

for i in location:
    for j in year:
        for data in rain['{0}_{1}'.format(i,j)]:
            if data not in rain_forecast['{0}_forecast_{1}'.format(i,j)]:
                incorrect += 1
            #실제 관측결과에선 강수가 있었지만 예보가 없던 날의 수만큼 변수에 더함
        for data_forecast in rain_forecast['{0}_forecast_{1}'.format(i,j)]:
            if data_forecast not in rain['{0}_{1}'.format(i,j)]:
                incorrect += 1
            #예보에선 강수가 있을것이라고 했지만 관측결과 강수가 없던 날의 수만큼 변수에 더함
                
accuracy = (1 - incorrect/full_date) * 100

결론¶

전체 영통구의 3년간의 일기 예보 정확도를 계산해본 결과 의 값이 산출되었다. 이는 내가 설정한 가설의 70%보다 낮은 수치이므로
나의 가설은 옳은 가설이라는 결론이 도출되었다.
특히 연도별로 나누어 계산해본 결과 연도에 따른 큰 차이가 보였는데 2018년의 예보 정확도는 70%넘는 상당히 정확한 수준을 이루었지만
2017년의 예보 정확도는 50%조금 아래쪽으로 계산 되었다. 이러한 차이가 발생하는 이유는 기상청이 데이터를 연도별로 저장하고 제공하였는데
이 과정에서 차이가 났다고 볼 수 있을 것이다.
또한 지역별로 나누어 분석해본 결과 대부분 비슷한 정확도를 보였지면 영통과 광교지역이 다른 지역보다 조금 낮은 수치를 보였다.
